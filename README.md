# IoT Device Penetration Testing Lab

## Overview

The **IoT Device Penetration Testing Lab** is a cybersecurity research project designed to analyze and identify security vulnerabilities in Internet of Things (IoT) devices such as smart cameras, smart bulbs, routers, and IoT modules.

This lab simulates a real-world IoT environment where a penetration tester analyzes network communication, firmware security, authentication mechanisms, and device services.

The project demonstrates how attackers can identify weaknesses in IoT ecosystems and provides mitigation strategies to improve device security.

---

## Objectives

* Build an isolated IoT security testing environment
* Discover IoT devices in a network
* Capture and analyze IoT traffic
* Extract and analyze firmware
* Identify vulnerabilities in IoT devices
* Demonstrate exploitation techniques
* Document findings and mitigation strategies

---

## Lab Architecture

The lab consists of an isolated network where multiple IoT devices are connected to a dedicated router. A Kali Linux machine acts as the attacker system and performs penetration testing.

## Lab Experiments

### 1. Device Discovery

Identify IoT devices connected to the network.

Techniques used:

* Network scanning
* Service detection
* Device fingerprinting

Example command:

```
nmap -sV 192.168.10.0/24
```

---

### 2. Traffic Analysis

Capture and analyze network traffic generated by IoT devices.

Focus areas:

* HTTP communication
* MQTT traffic
* DNS queries
* Authentication tokens

Example filter:

```
ip.addr == 192.168.10.10
```

---

### 3. Firmware Analysis

Extract and analyze firmware to identify hidden vulnerabilities.

Steps:

1. Download firmware
2. Extract using Binwalk
3. Analyze files and binaries

Example command:

```
binwalk -e firmware.bin
```

---

### 4. Vulnerability Testing

Test IoT devices for security weaknesses such as:

* Default credentials
* Open ports
* Unencrypted communication
* Insecure web interfaces

Example:

```
hydra -l admin -P passwords.txt 192.168.10.10 http-get
```

## Vulnerabilities Identified

| Device       | Vulnerability                  | Severity |
| ------------ | ------------------------------ | -------- |
| Smart Camera | Default Credentials            | High     |
| Smart Bulb   | Unencrypted MQTT Communication | Medium   |
| IoT Router   | Open Telnet Service            | High     |
| Smart Plug   | Weak Web Authentication        | Medium   |

---

## Mitigation Strategies

Recommended security improvements:

* Change default credentials
* Disable unnecessary services
* Implement HTTPS encryption
* Secure MQTT communication with TLS
* Regular firmware updates
* Disable Telnet access
